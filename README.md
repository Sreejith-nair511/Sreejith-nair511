## Engineering Principles

<div align="center">

<table>
<tr>
<td width="50%" valign="top">

**On Code Quality**
```
01  Read before writing.
    Understand the system first.

02  Name things precisely.
    A bad name is a silent bug.

03  One responsibility per unit.
    Functions should do one thing well.

04  Errors are values, not afterthoughts.
    Handle them at the boundary.
```

</td>
<td width="50%" valign="top">

**On Performance and Design**
```
05  Measure before optimizing.
    Gut feel about perf is usually wrong.

06  The best abstraction is often none.
    Add layers only when they earn their cost.

07  Docs explain why, not what.
    The code already says what.

08  Boring tech for boring problems.
    Save clever for when it counts.
```

</td>
</tr>
</table>
